# 20240427: API와 이벤트 핸들링의 기본 원리 고민

## 💡 Key Context
> 오늘 접한 키워드는 API와 이벤트 핸들링이었음. 특히 API의 요청-응답 구조와 Webhook이 어떻게 흐르는지에 관심이 갔고, 이를 실무에 어떻게 적용할지 고민하면서 중요도를 느꼈음.

## 🔍 Deep Dive & Reasoning
- API는 기본적으로 클라이언트와 서버 간에 데이터를 주고받기 위한 표준화된 인터페이스라고 이해했음. RESTful API의 경우, 요청과 응답의 명확한 규칙이 있기에 오케이인 듯함.
- 이벤트 핸들링은 어떤 특정 액션이 일어났을 때 이를 감지하고 적절히 반응하는 구조인 듯함. Webhook은 서버에서 발생한 이벤트를 외부로 즉시 전달하는 수단으로 이해됨.
- 왜 이렇게 설계했을까? 요청-응답 구조는 명확성과 확장성을 위해서인 것 같음. 이벤트 핸들링은 느슨한 결합과 실시간 처리 가능성을 높이기 위한 것으로 보임.
- 하지만, 여러 API 호출이 병목 현상이나 실패 시 어떻게 재시도 로직을 넣을지 고민이 됨. 또, 이벤트 핸들링에서는 이벤트 드리븐 아키텍처의 장단점을 고민했음.

## 🛠️ Troubleshooting & Insight
- 처음에는 Webhook URL 설정이나 trigger 조건이 막막했지만, 실제 사용할 API 문서를 보면서 구조를 그려보니 조금 감이 왔음.
- 잘못된 요청이나 응답 실패 시 어떻게 대응하는 프로토콜이 되는지 예시를 찾아보면서 개선 로직도 한번 정리해봐야겠다는 생각이 들었음.
