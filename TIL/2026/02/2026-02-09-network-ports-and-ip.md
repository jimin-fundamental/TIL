# 20260209: 네트워크 포트, IP 개념 정리와 웹 통신 구조 이해

## 💡 Key Context
> 오늘은 네트워크의 기본 개념과 포트, IP의 종류, 그리고 HTTP와 웹 통신에 대해 정리했음. 특히, 포트 구분과 IP의 차이, 그리고 HTTP 특징과 세션 구조가 나에게 중요하게 다가왔음. 실무에서 네트워크 기초 이해 없으면 문제 해결이 어려울 것 같아 이 부분에 집중하려 함.

## 🔍 Deep Dive & Reasoning
- 포트 수가 65,536개이고, Well-known 포트, Registered 포트, 그리고 Dynamic 포트로 구분되며, 각각 용도가 다름을 다시 확인. 이 구조가 왜 이렇게 되어 있는지 자바의 서드파티 라이브러리 동작 방식을 떠올리며, 다양한 서비스들이 포트 번호 구분으로 자신들의 트래픽을 구별하는 것임을 유추.
- Public IP와 Private IP의 차이는 명확했음. 공인 IP는 전 세계 고유 주소이고, 사설 IP는 내부 네트워크용이니, 라우터와 NAT의 역할이 딱 맞아떨어졌음. 특히 NAT의 존재가 사설 IP 부족 문제와 보안 강화에 얼마나 크게 기여하는지 갑자기 이해됨.
- HTTP는 stateless 하고 connectionless하며, 쿠키와 세션이 어떻게 다른지 확실히 구별 필요. SSE는 지속적 연결인데, 전통적 의미의 세션과 다르다는 점이 헷갈렸는데, 실시간 데이터 푸시에 초점 맞춰 생각하니 이해가 됨.
- TLS와 SSL의 차이, 그리고 현재 TLS가 더 표준이라는 사실이 기억에 남. 보안이 중요한데, HTTP와는 별도로 암호화 프로토콜이 존재함을 다시 떠올림.
- REST와 REST API의 차이 역시 중요한 개념임. REST는 원칙, REST API는 그 구현체라는 것을 정리.

## 🛠️ Troubleshooting & Insight
- 오늘 노트 정리하면서 포트와 IP 개념이 곧 실무에 어떻게 적용되는지 떠올리니, 막연히 아는 것보다 훨씬 첨단 지식이 된 느낌. 특히 NAT와 사설 IP의 역할이 왜 중요한지 실무 경험과 결부하여 생각하려 고심했음.
- 기술 구글링하면서 예전보다 속도도 붙고, 개념도 명확히 잡힌 것 같아 성취감이 큼. 실습과 동시에 이론을 정리하는 방식이 효과적임.
