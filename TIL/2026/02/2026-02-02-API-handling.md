# 20260202: API 요청과 응답 처리 방식에 대한 고민

## 💡 Key Context
> 오늘은 새로운 프로젝트에서 API와의 연동을 시작하면서, 요청과 응답 처리가 왜 이렇게 중요한지 실감하게 됐음. REST API를 통해 데이터를 주고받는 방식이 익숙하지 않아서 어떤 구조로 설계하는 게 효율적일지 고민했음.

## 🔍 Deep Dive & Reasoning
- API 가이드라인을 보면서 요청 구조, 헤더, 파라미터, 응답 포맷들을 이해하려 했음.
- 결국, 일관된 형식을 유지하는 게 유지보수에 유리하겠다는 개인적 추측이 떠올랐음. 특히, JSON 포맷을 표준으로 삼고, 오류처리 방식도 통일하는 게 중요하다고 느꼈음.
- 하지만, 깔끔한 아키텍처를 위해 비동기 요청처리, 에러 핸들링, 타임아웃 같은 부분에서 어떤 trade-off가 있는지 고민했음. 예를 들어, async/await로 처리하면 코드가 깔끔해지지만, 적절한 에러 핸들링이 필수임.

## 🛠️ Troubleshooting & Insight
- 처음엔 API 스펙 문서를 그대로 따라가면서 복잡함을 느꼈는데, 실제 요청 테스트를 통해 필요한 필드들만 쿼리하거나, 응답 형식을 단순화하는 방향으로 수정했더니 작업이 좀 더 수월했음.
- 서버에서 응답 헤더에 캐시 정책이나 CORS 정책이 어떻게 작동하는지 일부 실수로 오해했었지만, 팀 피드백을 받고 그 구조를 다시 이해함. 세부 정책 차이를 명확히 파악하는 게 우선임을 깨달았음.

## 🎯 Next Step
- 다양한 API 요청 테스트 과정에서 발생하는 에러 핸들링 방법을 좀 더 심화해서 공부할 예정임. 또한, 비동기로 처리하는 경우의 데이터 흐름에 대해 심층적으로 이해할 필요가 있음.
