# 20260207: HTTPS의 TLS Handshake와 인증서 발급 프로세스 이해

## 💡 Key Context  
> 오늘은 네트워크 기본 흐름과 HTTPS의 핵심인 TLS Handshake, 인증서 발급 프로세스에 대해 정리했음. 특히 TLS의 대칭키와 비대칭키의 역할, 인증서 검증 프로세스가 왜 중요한지 다시 생각하게 됐음.

## 🔍 Deep Dive & Reasoning  
- TLS Handshake는 대칭키의 효율성과 비대칭키의 보안성을 결합하는 과정이란 설명이 이해가 됨. 인증서 검증을 통해 서버의 신원을 확보하는 것은 HTTPS 보안의 핵심이라는 점 때문에, 브라우저와 서버 간 인증 과정이 왜 필요한지 자연스럽게 연결됨.
- 1.2버전보다 속도가 개선된 TLS 1.3은 왕복횟수와 보안 알고리즘 변화 덕에 성능과 안전 모두 향상된 것 같음. 그러나 이러한 변경이 실무에 어떤 영향을 미치는지 더 깊게 공부할 필요가 있음.
- 인증서 발급 과정에서 서버의 개인키는 절대 외부에 공개하지 않는다는 점이 중요함. CA의 서명, 신뢰의 사슬 구조를 떠올리면 자연스럽게 신뢰체계의 복잡성과 장단점이 느껴졌음.

## 🛠️ Troubleshooting & Insight  
- CSR 과정에서 서버 관리자는 공개키와 도메인 정보만 보내고, 개인키는 서버 내부에 안전하게 보관하는 게 핵심임. 이를 잊지 않기 위해 명확하게 기억하는 습관이 필요하다고 느낌.
- 인증서의 신뢰 사슬과 CA의 역할 이해가 좀 더 명료해질 필요 같음. 아직은 각 인증서와 CA의 역할이 구체적으로 어떻게 연결되는지 그림을 그려보는 연습이 필요함.
